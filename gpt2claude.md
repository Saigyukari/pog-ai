- `train_bc.py` now implements the roadmap BC launcher instead of the earlier smoke-test wrapper.
- Verified locally on a 128-record subset: 1 epoch completed and wrote `/tmp/pog_bc_subset/epoch_001.pkl`.
- In this environment JAX could not initialize CUDA, so validation ran on CPU fallback rather than multi-GPU `pmap`.
- Phase 2 DONE: `src/env/jax_env.py` and `src/rl/replay_buffer.py` written; `mcts.py` patched (MAX_NODES=512, depth_limit=4).
- Important dtype note: the roadmap says `int8` card indices, but the repo has 130 unique cards. JAX env therefore uses `int16` hands/decks with `255` sentinel because `int8` cannot represent all card ids safely.
- Scope note: the current Python baseline env still has no populated starting units and stubbed card-event resolution. `jax_env.py` intentionally matches that baseline rather than inventing unsupported full-game rules.
- Storage note: `replay_buffer.py` implements HDF5 save/load behind an optional `h5py` import; `h5py` is not installed in this environment, so hot-path buffer tests cover push/sample only.
- RL note: `VRAMReplayBuffer` now stores `policy` targets as well as sampled `action`, because `train_selfplay.py` trains against MCTS visit-count distributions, not one-hot action labels.
- `train_selfplay.py` now accepts the hardware-oriented CLI (`--n-actors`, `--batch-size`, `--checkpoint-in`) and applies the staged search schedule from `design_doc.md §2a` via `get_search_params(total_games_played)`.
- `eval/tournament.py` now exists, uses the CPU env + Python `MCTS` for offline checkpoint matches, anchors `random` at Elo 0, and smoke-ran successfully with `/tmp/selfplay_seed.pkl`.
- `README.md` quick-start is fixed: no `setup_env.sh` reference, and cluster/local BC commands now match the reviewed config.
- `play.py` now exists and smoke-ran against `/tmp/selfplay_seed.pkl`; it uses `pog_env.py` + Python `MCTS`, groups legal actions for human input, and exits cleanly on EOF when stdin closes.
- `src/data/starting_positions.py` now exists and parses `data/data.js` into shared initial unit arrays; `jax_env.py` and `pog_env.py` both consume it, so reset no longer starts from an empty board.
- Verified locally: `UNIT_FACTION_INIT[1] == CP`, `UNIT_FACTION_INIT[32] == AP`, and `jax_legal_mask(jax_reset(...))` now includes MOVE_UNIT actions. `tests/test_starting_positions.py` passes.
- `jax_env.py` now recomputes VP from control changes and returns non-zero terminal reward; reset starts from the initial board-control VP instead of hardcoded 0.
- Verified locally: `jax_reset(...).vp` is `-1` on the current starting map, terminal `jax_step` with `vp=25` returns non-zero reward, and `tests/test_vp_tracking.py` passes.
- `jax_env.py` now re-deals both hands at each turn boundary via `_deal_hand_from_deck` and `_advance_turn(state, deal_key)`, so games do not become cardless after turn 1.
- Verified locally: reset gives AP/CP hands of size 6/7, and forcing `action_round=7` then stepping with PASS wraps to turn 2 with both hands re-filled. `tests/test_card_deal.py` passes.
- `jax_env.py` no longer collapses war status into a single field. `JaxGameState` now carries `war_status_ap` and `war_status_cp`, `jax_legal_mask()` gates cards against the active player's own phase state, and `jax_obs()` exposes separate AP/CP total-war planes again. This restores parity with `pog_engine.py` / `pog_env.py` observations.
- `pog_env.py` now initializes and recomputes VP from board control instead of leaving `vp_track` stale at 0. CPU-env terminal winner logic, `play.py`, and `eval/tournament.py` were updated to the same VP sign convention used by `jax_env.py`: positive = AP winning, negative = CP winning.
- Verified locally: `python -m py_compile ...` passed and `pytest -q tests/test_jax_env.py tests/test_vp_tracking.py tests/test_war_status_alignment.py tests/test_pog_env_redeal.py` passed (`8 passed`).
- `jax_env.py` now advances war status at turn boundaries: on wrap, `vp >= +5` promotes AP from Limited→Total, and `vp <= -5` promotes CP. The transition is one-way and only checked on turn wrap, which keeps the starting `vp=-1` board from unlocking Total War immediately.

- `jax_env.py` now distinguishes permanent discard from normal card use. Static card tables include `CARD_REMOVE_AFTER_EVENT`, `_remove_active_card()` takes a `permanent` flag, event play uses the card metadata, and OPS play always leaves reusable cards out of the permanent discard mask so they can re-enter future hands.
- Added `tests/test_discard_fix.py`; verified with `python -m py_compile src/env/jax_env.py tests/test_discard_fix.py` and `pytest -q tests/test_discard_fix.py tests/test_jax_env.py tests/test_card_deal.py tests/test_war_status_advance.py tests/test_war_status_alignment.py` (`13 passed`).
- `jax_env.py` now exposes `CARD_REINF_COUNT` and `_bring_units_on_map()`. Playing a reinforcement-named card as an event brings one OFFBOARD unit of the active faction onto the first friendly source space, which starts filling the board with reserve units instead of leaving 177/194 units permanently off-map.
- `jax_env.py` now routes OPS subtype `2` (SR) through `_bring_units_on_map()`, so SR no longer behaves like a no-op card discard. The current simplification is one SR action -> one unit moved from OFFBOARD to a source space.
- Added `tests/test_reinforcement_event.py` and `tests/test_sr_event.py`; verified with `python -m py_compile src/env/jax_env.py tests/test_reinforcement_event.py tests/test_sr_event.py` and `pytest -q tests/test_reinforcement_event.py tests/test_sr_event.py tests/test_discard_fix.py tests/test_jax_env.py tests/test_card_deal.py tests/test_war_status_advance.py tests/test_war_status_alignment.py` (`15 passed`).
